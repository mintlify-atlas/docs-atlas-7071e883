Pydantic is the core validation library used by [FastAPI](https://fastapi.tiangolo.com/), providing automatic request validation, response serialization, and interactive API documentation.

## Why Pydantic + FastAPI?

FastAPI leverages Pydantic models for:

- **Automatic request validation** - Parse and validate request bodies, query parameters, and path parameters
- **Response serialization** - Convert Python objects to JSON with proper type handling
- **API documentation** - Generate OpenAPI/Swagger docs automatically from your models
- **Type safety** - Full editor support with type hints and autocompletion

## Basic Usage

### Request Body Validation

Pydantic models automatically validate incoming request data:

```python
from fastapi import FastAPI
from pydantic import BaseModel, EmailStr, Field

app = FastAPI()

class User(BaseModel):
    id: int
    username: str = Field(min_length=3, max_length=50)
    email: EmailStr
    full_name: str | None = None

@app.post("/users/")
async def create_user(user: User):
    # user is already validated and typed
    return {"id": user.id, "username": user.username}
```

FastAPI automatically:
- Validates the request body matches the `User` model
- Returns 422 validation errors for invalid data
- Generates API documentation showing the schema

### Response Models

Use Pydantic models to define response schemas with `response_model`:

```python
from typing import List
from pydantic import BaseModel

class UserResponse(BaseModel):
    id: int
    username: str
    email: str

    model_config = {"from_attributes": True}

@app.get("/users/{user_id}", response_model=UserResponse)
async def get_user(user_id: int):
    # Return data - FastAPI serializes it according to UserResponse
    return {"id": user_id, "username": "john_doe", "email": "john@example.com", "password": "secret"}
    # Note: 'password' field is automatically excluded from response
```

<Note>
The `response_model` parameter ensures only fields defined in the model are included in the response, providing automatic filtering.
</Note>

## Advanced Patterns

### Query Parameters and Path Validation

Use Pydantic models for complex query parameter validation:

```python
from typing import Annotated
from pydantic import BaseModel, Field
from fastapi import Query

class Pagination(BaseModel):
    skip: int = Field(default=0, ge=0)
    limit: int = Field(default=10, ge=1, le=100)

@app.get("/items/")
async def list_items(
    page: Annotated[int, Query(ge=1)] = 1,
    per_page: Annotated[int, Query(ge=1, le=100)] = 10
):
    return {"page": page, "per_page": per_page}
```

### Nested Models

Pydantic handles complex nested structures automatically:

```python
from typing import List
from pydantic import BaseModel

class Address(BaseModel):
    street: str
    city: str
    state: str
    postal_code: str

class Product(BaseModel):
    id: int
    name: str
    price: float = Field(gt=0)
    description: str | None = None

class Order(BaseModel):
    order_id: str
    customer_email: EmailStr
    shipping_address: Address
    products: List[Product]

@app.post("/orders/", response_model=Order)
async def create_order(order: Order):
    # All nested models are validated automatically
    return order
```

### Field Validation with Validators

```python
from pydantic import BaseModel, field_validator

class BlogPost(BaseModel):
    title: str
    content: str
    author: str
    published: bool = False

    @field_validator('title')
    @classmethod
    def title_must_be_capitalized(cls, v: str) -> str:
        if not v[0].isupper():
            raise ValueError('Title must start with a capital letter')
        return v

    @field_validator('content')
    @classmethod
    def content_must_be_long_enough(cls, v: str) -> str:
        if len(v) < 50:
            raise ValueError('Content must be at least 50 characters')
        return v

@app.post("/posts/")
async def create_post(post: BlogPost):
    return post
```

## Performance Optimization

### Type Adapters for Repeated Validations

For endpoints that validate the same type repeatedly, FastAPI creates a `TypeAdapter` once and reuses it:

```python
from pydantic import TypeAdapter

# FastAPI does this internally for each endpoint
UserAdapter = TypeAdapter(User)

# Validation is optimized through compiled validators
user = UserAdapter.validate_python(data)
```

<Note>
Pydantic v2 compiles validators to Rust code via `pydantic-core`, providing significant performance improvements for FastAPI applications.
</Note>

### Response Serialization

Optimize JSON serialization with model configuration:

```python
from pydantic import BaseModel, ConfigDict

class OptimizedUser(BaseModel):
    model_config = ConfigDict(
        # Serialize by alias for API responses
        populate_by_name=True,
        # Use faster JSON serialization
        ser_json_timedelta='float',
    )
    
    id: int
    username: str
    created_at: datetime
```

## Error Handling

<Steps>

### Validation Errors

FastAPI automatically catches Pydantic `ValidationError` and returns 422 responses:

```python
from fastapi import FastAPI, HTTPException
from pydantic import ValidationError

app = FastAPI()

@app.post("/users/")
async def create_user(user: User):
    # Validation happens automatically before this function
    # Invalid data returns 422 with detailed error information
    return user
```

### Custom Error Messages

Provide user-friendly error messages:

```python
from pydantic import BaseModel, Field

class User(BaseModel):
    username: str = Field(
        min_length=3,
        max_length=50,
        description="Username must be between 3 and 50 characters"
    )
    age: int = Field(ge=0, le=150, description="Age must be between 0 and 150")
```

### Exception Handlers

Handle validation errors globally:

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from pydantic import ValidationError

app = FastAPI()

@app.exception_handler(ValidationError)
async def validation_exception_handler(request: Request, exc: ValidationError):
    return JSONResponse(
        status_code=422,
        content={"detail": exc.errors()},
    )
```

</Steps>

## JSON Schema Generation

FastAPI uses Pydantic's JSON Schema generation for OpenAPI documentation:

```python
from pydantic import BaseModel, Field

class Item(BaseModel):
    """An item in the inventory"""
    
    name: str = Field(description="The item's name")
    price: float = Field(gt=0, description="Price must be greater than zero")
    tags: list[str] = Field(default=[], description="Optional tags for categorization")

# Access the generated schema
schema = Item.model_json_schema()
# FastAPI uses this to generate OpenAPI/Swagger documentation
```

<Warning>
Be careful with circular references in models used with FastAPI. Use `model_rebuild()` after forward reference resolution.
</Warning>

## Best Practices

1. **Separate request and response models** - Use different models for input validation and output serialization
   ```python
   class UserCreate(BaseModel):  # For POST requests
       username: str
       password: str
       email: EmailStr

   class UserResponse(BaseModel):  # For responses
       id: int
       username: str
       email: str
       # No password field!
   ```

2. **Use Field() for documentation** - Add descriptions that appear in API docs
   ```python
   class User(BaseModel):
       username: str = Field(description="Unique username")
       email: EmailStr = Field(description="User's email address")
   ```

3. **Enable from_attributes for ORMs** - When working with database models
   ```python
   class UserResponse(BaseModel):
       model_config = ConfigDict(from_attributes=True)
       
       id: int
       username: str
   
   # Works with SQLAlchemy, Tortoise ORM, etc.
   db_user = session.query(DBUser).first()
   return UserResponse.model_validate(db_user)
   ```

4. **Leverage strict mode for APIs** - Prevent type coercion when needed
   ```python
   class StrictUser(BaseModel):
       model_config = ConfigDict(strict=True)
       
       id: int  # Only accepts int, not "123"
       active: bool  # Only accepts true/false, not 1/0
   ```
