The `pydantic.version` module holds the version information for Pydantic and provides utilities for version checking and information display.

## VERSION

```python
VERSION = '2.13.0b2'
```

The version of Pydantic.

This version specifier is guaranteed to be compliant with the [specification](https://packaging.python.org/en/latest/specifications/version-specifiers/) introduced by [PEP 440](https://peps.python.org/pep-0440/).

### Example

```python
from pydantic.version import VERSION

print(f"Using Pydantic version: {VERSION}")
# Using Pydantic version: 2.13.0b2
```

## version_short

```python
def version_short() -> str
```

Return the `major.minor` part of Pydantic version.

It returns '2.1' if Pydantic version is '2.1.1'.

<ResponseField name="return" type="str">
  The major.minor version string.
</ResponseField>

### Example

```python
from pydantic.version import version_short

print(version_short())
# '2.13'
```

## version_info

```python
def version_info() -> str
```

Return complete version information for Pydantic and its dependencies.

This function provides detailed information about:
- Pydantic version
- Pydantic-core version
- Pydantic-core build information
- Python version
- Platform information
- Related packages (fastapi, mypy, typing_extensions, etc.)
- Git commit (if applicable)

<ResponseField name="return" type="str">
  A formatted string containing complete version information.
</ResponseField>

### Example

```python
from pydantic.version import version_info

print(version_info())
# Output:
#              pydantic version: 2.13.0b2
#         pydantic-core version: 2.42.0
#           pydantic-core build: profile=release pgo=false
#                 python version: 3.11.5 (main, Sep 11 2023, 13:54:46)
#                       platform: Linux-5.15.0-1034-azure-x86_64-with-glibc2.35
#                related packages: fastapi-0.109.0 mypy-1.8.0 typing_extensions-4.9.0
#                         commit: unknown
```

## check_pydantic_core_version

```python
def check_pydantic_core_version() -> bool
```

Check that the installed `pydantic-core` dependency is compatible.

This function verifies that the installed version of `pydantic-core` matches the version required by the current Pydantic installation.

<ResponseField name="return" type="bool">
  `True` if the pydantic-core version is compatible, `False` otherwise.
</ResponseField>

### Example

```python
from pydantic.version import check_pydantic_core_version

if check_pydantic_core_version():
    print("pydantic-core version is compatible")
else:
    print("pydantic-core version mismatch")
```

## parse_mypy_version

```python
def parse_mypy_version(version: str) -> tuple[int, int, int]
```

Parse `mypy` string version to a 3-tuple of ints.

It parses normal version like `1.11.0` and extra info followed by a `+` sign like `1.11.0+dev.d6d9d8cd4f27c52edac1f537e236ec48a01e54cb.dirty`.

<ParamField path="version" type="str" required>
  The mypy version string.
</ParamField>

<ResponseField name="return" type="tuple[int, int, int]">
  A triple of ints representing the version, e.g. `(1, 11, 0)`.
</ResponseField>

### Example

```python
from pydantic.version import parse_mypy_version

version = parse_mypy_version('1.11.0')
print(version)
# (1, 11, 0)

version_dev = parse_mypy_version('1.11.0+dev.abc123.dirty')
print(version_dev)
# (1, 11, 0)
```

## Internal Constants

### _COMPATIBLE_PYDANTIC_CORE_VERSION

```python
_COMPATIBLE_PYDANTIC_CORE_VERSION = '2.42.0'
```

The compatible pydantic-core version required by this version of Pydantic.

This should be kept in sync with the version constraint in the `pyproject.toml` dependencies.

## Version Compatibility

Pydantic enforces strict version compatibility with pydantic-core. If you encounter a version mismatch error, you should ensure that both packages are updated together:

```bash
pip install -U pydantic pydantic-core
```

Or if using poetry:

```bash
poetry update pydantic pydantic-core
```

The version check is performed automatically when Pydantic is imported, and will raise a `SystemError` if incompatible versions are detected (except in editable/development installations).
