A base class for creating Pydantic models.

## Class Definition

```python
class BaseModel(metaclass=ModelMetaclass)
```

## Class Attributes

<ResponseField name="model_config" type="ClassVar[ConfigDict]">
Configuration for the model, should be a dictionary conforming to [`ConfigDict`](/api/config).
</ResponseField>

<ResponseField name="model_fields" type="dict[str, FieldInfo]">
A mapping of field names to their respective `FieldInfo` instances.

<Warning>
Accessing this attribute from a model instance is deprecated. Access it from the model class instead.
</Warning>
</ResponseField>

<ResponseField name="model_computed_fields" type="dict[str, ComputedFieldInfo]">
A mapping of computed field names to their respective `ComputedFieldInfo` instances.
</ResponseField>

## Instance Attributes

<ResponseField name="model_extra" type="dict[str, Any] | None">
Get extra fields set during validation. Returns `None` if `config.extra` is not set to `"allow"`.
</ResponseField>

<ResponseField name="model_fields_set" type="set[str]">
Returns the set of fields that have been explicitly set on this model instance.
</ResponseField>

## Constructor

### `__init__`

```python
def __init__(self, /, **data: Any) -> None
```

Create a new model by parsing and validating input data from keyword arguments.

<ParamField path="data" type="Any">
Keyword arguments containing the data to validate.
</ParamField>

<ResponseField name="raises" type="ValidationError">

## Validation Methods

### `model_validate`

```python

def model_validate(
    cls,
    obj: Any,
    *,
    strict: bool | None = None,
    extra: ExtraValues | None = None,
    from_attributes: bool | None = None,
    context: Any | None = None,
    by_alias: bool | None = None,
    by_name: bool | None = None,

```

Validate a pydantic model instance.

<ParamField path="obj" type="Any">
The object to validate.
</ParamField>

<ParamField path="strict" type="bool | None">
Whether to enforce types strictly.
</ParamField>

<ParamField path="extra" type="ExtraValues | None">
Whether to ignore, allow, or forbid extra data during model validation.
</ParamField>

<ParamField path="from_attributes" type="bool | None">
Whether to extract data from object attributes.
</ParamField>

<ParamField path="context" type="Any | None">
Additional context to pass to the validator.
</ParamField>

<ParamField path="by_alias" type="bool | None">
Whether to use the field's alias when validating against the provided input data.
</ParamField>

<ParamField path="by_name" type="bool | None">
Whether to use the field's name when validating against the provided input data.
</ParamField>

<ResponseField name="return" type="Self">
The validated model instance.
</ResponseField>

<ResponseField name="raises" type="ValidationError">

</ResponseField>


## Serialization Methods

### `model_dump`

```python
def model_dump(
    self,
    *,
    mode: Literal['json', 'python'] | str = 'python',
    include: IncEx | None = None,
    exclude: IncEx | None = None,
    context: Any | None = None,

    exclude_unset: bool = False,

    fallback: Callable[[Any], Any] | None = None,
    serialize_as_any: bool = False,
    polymorphic_serialization: bool | None = None,
) -> dict[str, Any]
```

Generate a dictionary representation of the model.

<ParamField path="mode" type="Literal['json', 'python'] | str" default="'python'">
The mode in which to serialize. If 'json', only JSON-serializable types. If 'python', may contain Python objects.
</ParamField>

<ParamField path="include" type="IncEx | None">
A set of fields to include in the output.
</ParamField>

<ParamField path="exclude" type="IncEx | None">
A set of fields to exclude from the output.
</ParamField>

<ParamField path="context" type="Any | None">
Additional context to pass to the serializer.
</ParamField>

<ParamField path="by_alias" type="bool | None">
Whether to use the field's alias in the dictionary key if defined.
</ParamField>

<ParamField path="exclude_unset" type="bool" default="False">
Whether to exclude fields that have not been explicitly set.
</ParamField>

<ParamField path="exclude_defaults" type="bool" default="False">
Whether to exclude fields that are set to their default value.
</ParamField>

<ParamField path="exclude_none" type="bool" default="False">
Whether to exclude fields that have a value of `None`.
</ParamField>

<ParamField path="exclude_computed_fields" type="bool" default="False">
Whether to exclude computed fields.
</ParamField>

<ParamField path="round_trip" type="bool" default="False">
If True, dumped values should be valid as input for non-idempotent types.
</ParamField>

<ParamField path="warnings" type="bool | Literal['none', 'warn', 'error']" default="True">
How to handle serialization errors. False/"none" ignores them, True/"warn" logs errors, "error" raises.
</ParamField>

<ParamField path="fallback" type="Callable[[Any], Any] | None">
A function to call when an unknown value is encountered.
</ParamField>

<ParamField path="serialize_as_any" type="bool" default="False">
Whether to serialize fields with duck-typing serialization behavior.
</ParamField>

<ParamField path="polymorphic_serialization" type="bool | None">
Whether to use model and dataclass polymorphic serialization.
</ParamField>

<ResponseField name="return" type="dict[str, Any]">
A dictionary representation of the model.
</ResponseField>

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int = 0

user = User(name='John', age=30)
print(user.model_dump())
#> {'name': 'John', 'age': 30}
print(user.model_dump(exclude_defaults=True))
#> {'name': 'John', 'age': 30}
```

## Construction Methods

### `model_construct`

```python
@classmethod
def model_construct(
    cls,
    _fields_set: set[str] | None = None,
    **values: Any
) -> Self
```

Creates a new instance of the Model class with validated data.

Creates a new model setting `__dict__` and `__pydantic_fields_set__` from trusted or pre-validated data. Default values are respected, but no other validation is performed.

<ParamField path="_fields_set" type="set[str] | None">
A set of field names that were originally explicitly set. If provided, this is directly used for the `model_fields_set` attribute.
</ParamField>

<ParamField path="values" type="Any">
Trusted or pre-validated data dictionary.
</ParamField>

<ResponseField name="return" type="Self">
A new instance of the Model class with validated data.
</ResponseField>

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int = 0

user = User.model_construct(name='John', age=30)
print(user)
#> name='John' age=30
```

### `model_copy`

```python
def model_copy(
    self,
    *,
    update: Mapping[str, Any] | None = None,
    deep: bool = False
) -> Self
```

Returns a copy of the model.

<ParamField path="update" type="Mapping[str, Any] | None">
Values to change/add in the new model. Note: the data is not validated before creating the new model.
</ParamField>

<ParamField path="deep" type="bool" default="False">
Set to True to make a deep copy of the model.
</ParamField>

<ResponseField name="return" type="Self">
New model instance.
</ResponseField>

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int

user = User(name='John', age=30)
user_copy = user.model_copy(update={'age': 31})
print(user_copy)
#> name='John' age=31
print(user)
#> name='John' age=30
```

```python
@classmethod
def model_json_schema(
    cls,

```

Generates a JSON schema for a model class.

<ParamField path="by_alias" type="bool" default="True">
Whether to use attribute aliases or not.
</ParamField>

<ParamField path="ref_template" type="str">
The reference template.
</ParamField>

<ParamField path="union_format" type="Literal['any_of', 'primitive_type_array']" default="'any_of'">
The format to use when combining schemas from unions together.

<ParamField path="schema_generator" type="type[GenerateJsonSchema]">

```python
    age: int

```

### `model_rebuild`

```python
@classmethod
def model_rebuild(
    cls,
    *,
    force: bool = False,
    raise_errors: bool = True,
    _parent_namespace_depth: int = 2,
    _types_namespace: MappingNamespace | None = None,
) -> bool | None
```

Try to rebuild the pydantic-core schema for the model.

This may be necessary when one of the annotations is a ForwardRef which could not be resolved during the initial attempt to build the schema.

<ParamField path="force" type="bool" default="False">
Whether to force the rebuilding of the model schema.
</ParamField>

<ParamField path="raise_errors" type="bool" default="True">
Whether to raise errors.
</ParamField>

<ParamField path="_parent_namespace_depth" type="int" default="2">
The depth level of the parent namespace.
</ParamField>

<ParamField path="_types_namespace" type="MappingNamespace | None">
The types namespace.
</ParamField>

<ResponseField name="return" type="bool | None">
Returns `None` if the schema is already "complete". If rebuilding was required, returns `True` if successful, otherwise `False`.
</ResponseField>

```python
from __future__ import annotations
from pydantic import BaseModel

class User(BaseModel):
    name: str
    friend: User | None = None

# Rebuild to resolve the forward reference
User.model_rebuild()
```

## Lifecycle Hooks

### `model_post_init`

```python
def model_post_init(self, context: Any, /) -> None
```

Override this method to perform additional initialization after `__init__` and `model_construct`.

This is useful if you want to do some validation that requires the entire model to be initialized.

<ParamField path="context" type="Any">
The context passed to the model during initialization.
</ParamField>

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int
    
    def model_post_init(self, context):
        print(f'User {self.name} created!')

user = User(name='John', age=30)
#> User John created!
```
